# Code of Conduct

## Our Commitment

The goal of this project is to build a welcoming, respectful, and collaborative community for everyone.

We encourage constructive discussions, thoughtful feedback, and professional collaboration regardless of experience level, background, or area of expertise.

---

# Expected Behavior

Please:

* Be respectful and considerate.
* Assume good intentions.
* Provide constructive feedback.
* Focus discussions on technical topics.
* Help create a positive environment for contributors.
* Respect differing opinions and approaches.

---

# Unacceptable Behavior

The following behaviors are not acceptable:

* Harassment or personal attacks.
* Discrimination of any kind.
* Offensive, abusive, or insulting language.
* Trolling or intentionally disruptive behavior.
* Publishing private information without permission.
* Spam or unrelated promotional content.

---

# Communication

Technical disagreements are a normal part of software development.

When discussing implementations:

* Critique ideas, not people.
* Explain the reasoning behind your suggestions.
* Be open to alternative solutions.
* Keep conversations respectful, even when opinions differ.

---

# Pull Requests and Reviews

Reviews should aim to improve the project.

When reviewing code:

* Be constructive.
* Explain requested changes.
* Avoid dismissive comments.
* Respect that different solutions may be valid.

When receiving feedback:

* Assume it is intended to improve the project.
* Discuss technical trade-offs respectfully.
* Ask questions if something is unclear.

---

# Scope

This Code of Conduct applies to:

* GitHub Issues
* Pull Requests
* Discussions
* Project documentation
* Any other public interactions related to this repository

---

# Enforcement

Project maintainers may remove comments, issues, discussions, or contributions that violate this Code of Conduct.

Repeated or severe violations may result in temporary or permanent exclusion from participation in the project.
